Документация Rest API

1. Получение заявки

	GET /order?id={номер заявки}

	Ответ: 
	{
		"id": номер заявки,
		"name": "название фирмы",
		"driver": "название перевозчика",
		"phone": "телефон",
		"ati": номер ати,
		"comments": "комментарии",		
	}
	
	Название фирмы и прочие поля пусты, если id неправильный.

2. Добавление заявки
	POST /order
	
	Отправляемые данные
	{
		"name": "название фирмы",
		"driver": "название перевозчика",
		"phone": "телефон",
		"ati": номер ати,
		"comments": "комментарий",		
		"pass": "пароль для входа"		
	}
	
	Ответ: 
	{
		"result": номер ответа
	}
	0 - успешно
	1 - неправильный пароль
	2 - ошибка в данных (например, одно из полей пустое)

3. Редактирование заявки
	PUT /order
	
	Отправляемые данные
	{
		"id": номер заявки,
		"name": "новое название фирмы",
		"driver": "новое название перевозчика",
		"phone": "новый телефон",
		"ati": новый номер ати,
		"comments": "новый комментарий",		
		"pass": "пароль для входа"		
	}
	
	Ответ: 
	{
		"result": номер ответа
	}
	0 - успешно
	1 - неправильный пароль
	2 - неправильный id
	3 - ошибка в данных (например, одно из полей пустое)

4. Удаление заявки

	DELETE /order
	
	Отправляемые данные
	{
		"id": номер заявки,
		"pass": "пароль для входа"
	}
	
	Ответ: 
	{
		"result": номер ответа
	}
	0 - успешно
	1 - неправильный пароль
	2 - неправильный id
